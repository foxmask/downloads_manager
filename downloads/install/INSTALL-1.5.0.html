Version :
1.5.0
Auteur :
FoxMaSk
Date :
22/02/2010

Description

Basiquement, ce module permet de mettre en téléchargement des fichiers.
Mais au delà de cela, il convient très bien à des sites hébergeant plusieurs projets.
En effet les auteurs de projets peuvent utiliser ce module pour mettre en téléchargement les nouvelles versions de leur création.
Chaque auteur peut décider d'autoriser des visiteurs à télécharger ses fichiers ou bien uniquement les membres de son site.
Il peut également permettre (ou non) de télécharger ses fichiers depuis un site distant.

Installation

0) Structure du répertoire downloads

classes

  1. download_config.class.php : classe lisant le fichier de configuration du module
  2. download_files.class.php : classe gerant l'accès aux fichiers
  3. readmodule.class.php : classe lisant le fichier module.xml
  4. downloadsadmin.listener.php : classe generant le menu d'admin du module downloads

controllers

  1. default.classic.php : contrôleur pour l'accès publique
  2. feeds.classic.php : contrôleur générant les flux RSS
  3. mgr.classic.php : contrôleur de l'accès admin, gestion du module.

daos

  1. downloads.dao.xml : DAO de la table Downloads
  2. downloads_cat.dao.xml : DAO de la table Downloads Cat

forms

  1. config.form.xml : formulaire de configuration du module downloads
  2. edit.form.xml : formulaire d'edition/ajout de downloads

install

  1. files : dossier contenant les fichiers téléchargeables (cf plus bas)
  2. themes : dossier contenant images et css du module (cf plus bas)
  3. INSTALL-1.4.0.html : ce fichier ;)
  4. downloads.module.ini.php.dist : fichier de configuration (d'exemple) du module
  5. urls.xml : fichier contenant les urls à utiliser quand vous utilisez le moteur d'url significant

locales

  1. en_EN : traductions anglaise du module
  2. fr_FR : "V.O" du module ;)

templates

  1. en_EN : template en version anglaise
  2. fr_FR : template en "V.O" du module ;)

zones

  1. admin.zone.php : zone affichant le contenu du fichier module.xml
  2. admin_list.zone.php : zone affichant la liste des fichiers téléchargeables
  3. admincat.zone.php : zone affichant la liste catégories de téléchargements
  4. adminconfig.zone.php : zone affichant le formulaire de configuration du module
  5. list.zone.php : zone affichant la liste des téléchargements
  6. view.zone.php : zone affichant UN téléchargement

autres

  1. module.xml : fichier de définition du module au format jelix
  2. events.xml : fichier de gestion des evenements (necesasire pour le module master_module)

1) Fichiers

  • 1.1) Décompresser l'archive dans le dossier modules
  • 1.2) Copier les dossiers "install/themes" et "install/files" dans votre dossier www
  • 1.3) Copier le fichier install/downloads.module.ini.php.dist dans le dossier var/config/ sous le nom downloads.module.ini.php

2) Paramètrage :

  • 2.0) fichier downloads.module.ini.php
    changer la ligne suivante 
    commons.upload.dir=files

    en mettant le répertoire de stockage de vos fichiers relatif à votre application.
    Exemple indiquer commons.upload.dir=../files si le dossier files se trouve au meme niveau que votre application
  • 2.1) fichier defaultconfig.ini.php
  • 2.1.1) Assurez vous que le module downloads soit bien dans le modulePath (cf paramètre modulesPath dans var/config/defaultconfig.ini.php)
  • 2.1.2) dans le fichier config.ini.php de votre entrypoint par defaut, dans le paragraphe [simple_urlengine_entrypoints], ajouter ce qui suit : 
    downloads~mgr:index@classic downloads~mgr:manage@classic downloads~mgr:config@classic downloads~mgr:dls@classic

    sur la ligne contenant le point d'entrée à l'administration de votre site, exemple : 
    manager="master_admin~*@classic downloads~mgr:index@classic downloads~mgr:manage@classic downloads~mgr:config@classic downloads~mgr:dls@classic jacl2db_admin~*@classic"
  • 2.1.3) moteur d'urls
    - si vous souhaitez utiliser le moteur d'urls "significant" (URL Significant) ajouter dans votre fichier www/.htaccess les lignes suivantes :
    	
	<IfModule mod_rewrite.c>
		RewriteEngine On
		RewriteBase /
		RewriteCond %{REQUEST_FILENAME} !-f
		RewriteCond %{REQUEST_FILENAME} !-d
		RewriteRule ^(.*)$ index.php/$1 [L,QSA]
	</IfModule>
    et copier le contenu du fichier install/urls.xml dans le fichier urls.xml de votre dossier var/config/
    et enfin ajouter la ligne 
        Options +Multiviews
    dans la config de votre serveur apache.

3) Base de données :

éxécuter le script downloads.create-table.mysql.sql

dans la table downloads_users, ajouter le nom d'un utilisateur, le nom du répertoire auquel il a accès, et le nom du projet (le meme nom que celui du répertoire)
Celui ci doit etre dans le repertoire défini au point 2.0 plus haut

4) jAcl2 :

éxécuter le script downloads.jacl2-data.mysql.sql pour la gestion des droits du module

5) Gestion des téléchargements

Si votre application gère déjà des fichiers à envoyer dans un répertoire existant alors vous devez :

5.1) ajouter à votre fichier events.xml l'event dans votre listener

5.2) Supprimez du fichier events.xml du module downloads l'event

5.3) Créer la methode "onDownloadGetHostingDirectory" dans votre listener en vous inspirant de la methode se trouvant dans la classe downloadsadminListener

tout ceci permet à votre application d'indiquer au module downloads trouver le repertoire hébergeant les fichiers

6) Accéder au module :

  • 6.1) Accédez à la partie publique du module comme suit :
  • 6.1.1) avec le moteur d'urls simple :
    http://votresite/index.php?module=downloads
  • 6.1.2) avec le moteur d'urls significant :
    http://votresite/index/downloads
  • 6.2) Accédez à la partie administration grâce au module master_admin

7) Fils RSS

dans le fichier downloads.module.ini.php, modifiez manuellement les paramètres suivants avec les valeurs qui vous concernent : 

	website.url=http://votresite.downloads/
	website.copyright=FoxMaSk 2008
	website.description=FoxMaSk 0wnZ
	website.author=FoxMaSk
	website.ttl=60
